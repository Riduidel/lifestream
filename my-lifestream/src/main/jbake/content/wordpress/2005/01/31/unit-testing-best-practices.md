# unit testing best practices

One more gem from [JavaRunner](http://javarunner.blogspot.com/2005/01/test-driven-development-useful.html) about testing best practices, which could cope well with some others (maybe well known, but I prefer to remember them well) :
* A good test is hard to go through.
* For each bug found, add a test.
* When a test fails, don't debug : add other tests to know why it fails.
