# Spurl ? Pas tant que ??a !

J'ai utilisé comme point d'entrée dans ma gestion des bookmarks sur le web {Spurl.net}. Seulement voilà, c'est vraiment pourri.
*  Pourri, parce qu'il importe les favoris existants dans une structure de dossiers non taggée.
* Pourri parce que ces bookmarks importés ne sont pas copiés dans {del.icio.us}, alors même que son intérêt majeur, selon moi, est de fournir une surcouche (ajoutant un moteur de recherche) à {del.icio.us}.
* Pourri encore parce que, pour que je puisse copier ces bookmarks dans {del.icio.us}, il faut que je les enlève de {Spurl.net} pour les y remettre. Ce qui va me prendre un certain temps.
* Pourri enfin parce que, pour supprimer ces bookmarks, je dois aller dans chaque dossier, en supprimer tous les bookmarks, puis supprimer le dossier (restons propres et ne dégradons pas l'environnement). Ce qui, encore une fois, est un processus plutôt long. 

Donc {Spurl.net} risque fort pour moi de vivre ses derniers jours. En fait, je prononcerai son acte de décès quand j'aurais écrit [mon script](http://nicolas.delsaux.free.fr/wordpress/index.php?p=126) {Ruby} de gestion de bookmarks.via [Nicolas Delsaux's posterous import script](http://riduidel.posterous.com/quest-ce-que-cest-que-ce-flux-qui-clignote)
